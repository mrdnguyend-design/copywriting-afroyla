# Module 5: Writing Craft & Quality Control

> **Mục tiêu:** Từ draft → polish → approve. The review system that ensures consistent quality.
> **Data source:** Ben/Daniel review patterns, Throssell-style auditor, quality checklist, post-mortem patterns.

## 5.1 The Ben & Daniel Dual-Review System

### How It Works

Every important email gets reviewed by 2 expert personas before sending:

| | **Ben Settle** (Marketing) | **Daniel Throssell** (Story) |
|---|---|---|
| **Focus** | Hook, personality, status, sales psychology | Scene, dialogue, show vs tell, pacing |
| **Key question** | "Does this stop the scroll?" | "Can I see, hear, smell this scene?" |
| **Strengths** | CTA power, urgency, positioning | Narrative arc, emotional timing |

### Ben's 5 Scoring Criteria

| Criterion | ⭐/5 | What it measures |
|-----------|------|-----------------|
| **Hook Power** | | Dòng đầu có stop the scroll? |
| **Personality** | | Persona có hiện diện? |
| **Status Positioning** | | Reader cảm thấy là insider? |
| **Sales Psychology** | | CTA irresistible? Urgency tự nhiên? |
| **Polarization** | | Đủ mạnh để alienate non-target? |

### Daniel's 5 Scoring Criteria

| Criterion | ⭐/5 | What it measures |
|-----------|------|-----------------|
| **Scene Construction** | | Đọc lên có thấy, nghe, ngửi? |
| **Dialogue** | | Conversations có real? |
| **Show vs Tell** | | Show emotion qua action? |
| **Emotional Pacing** | | Tension build và release đúng nhịp? |
| **Specificity** | | Chi tiết cụ thể (2:43 AM) hay chung chung? |

### Review Output Format

```
═══ BEN'S REVIEW ═══
Hook Power:        ⭐⭐⭐⭐ (4/5)
[Specific line improvements...]

═══ DANIEL'S REVIEW ═══
Scene Construction: ⭐⭐⭐⭐⭐ (5/5)
[Specific line improvements...]

═══ JOINT ASSESSMENT ═══
Overall: ⭐⭐⭐⭐ (A-)
Strengths: [Top 3]
Improvements: [Top 3]
Revised version: [if score < 4]
```

### Phase Gate

```
Draft V1 → Ben/Daniel Review → Apply 3-4 Refinements → V2
→ User Review → Proofread → Final HTML → Send
```

- **Average iterations to approve:** 2-4 rounds
- **Fastest:** 1 round (post-sale recap — "near-perfect first draft")
- **Slowest:** 7+ rounds (IWD Teaser #1 — new format, defining process)

## 5.2 Anti-AI Proofread (Throssell-Style Auditor)

### 8 AI-isms to Eliminate

| # | AI-ism | Fix |
|---|--------|-----|
| 1 | **Em dash overuse** (—) | Max 2-3 per email. Replace with periods, commas, parentheses. |
| 2 | **"Here's the thing" transitions** | Delete. Trust the structure. |
| 3 | **Brochure triplets** | 3 parallel adj/phrases → one strong statement |
| 4 | **"If I told you..." preambles** | Jump-cut into the scene |
| 5 | **"So that's the story" lazy bridges** | Surface insight plainly: "Silence doesn't protect you." |
| 6 | **Staccato repetition** | Same rhythm 5+ sentences in a row → vary cadence |
| 7 | **Sentimental restatements** | Already proven through scene → don't restate the lesson |
| 8 | **"Softening" lines** | "Universities overcorrect" dilutes "They told her to delete." |

### The Em Dash Test

Count em dashes in your draft. If > 3:

```
"She paused — collecting herself — before walking in — head high."
                                                                 
→ Replace:

"She paused. Collected herself. Walked in, head high."
```

### The "Press Release" Test

Read the email aloud. If ANY sentence sounds like:
- A press release
- A corporate announcement  
- A brochure
- A LinkedIn post

→ Rewrite in Michelle's voice: "How would she say this at her kitchen table?"

### Common Rewrites

| ❌ AI/Corporate | ✅ Michelle Voice |
|----------------|-------------------|
| "We are thrilled to announce..." | "Okay so I did something." |
| "Our carefully curated collection..." | "These bags? Girl." |
| "Don't miss this limited-time opportunity" | "When they're gone, they're gone" |
| "We believe in empowering women" | *[Don't say it — show it through the story]* |
| "Exclusive, premium, curated" | "Real. Built. Yours." |

## 5.3 Email Quality Checklist (8 Points)

### Pre-Send Quality Gate

| # | Check | Question |
|---|-------|---------|
| 1 | **Is it a story?** | Does it draw readers in, or just TELL them something? |
| 2 | **One person to one person?** | No "Hello everyone" — "Sis. Let me tell you." |
| 3 | **Pace then lead?** | Start with their world → lead to your offer? |
| 4 | **Show, don't tell?** | "Steam from his ears" > "The man was mad" |
| 5 | **One specific emotion?** | Can you name it? (laugh, cry, envy, inspire?) |
| 6 | **Demonstrate?** | Does a customer story SHOW the product solving a problem? |
| 7 | **Dimensionalize?** | Reader imagines life AFTER buying? |
| 8 | **Direct CTA?** | Clear next action? Not "dressed up with nowhere to go"? |

## 5.4 Logic Check (5-Point Pre-Send Validation)

Run this BEFORE presenting any draft:

- [ ] **Character consistency** — Recurring characters in-character? George oblivious-sweet, never mean?
- [ ] **Timeline logic** — Timestamps/events make sense? "By the time Dee and I were GROWN" (not childhood)?
- [ ] **Emotional arc** — vulnerability → revelation → pride → action? (not flat?)
- [ ] **Scene believability** — Could this happen in real life? No impossible logistics?
- [ ] **Voice check** — Sounds like Michelle texting a friend, or marketer writing a campaign?

### Specific Logic Traps (from post-mortems)

| Trap | Example | Fix |
|------|---------|-----|
| **False conditional** | "If you haven't shopped yet" (sent to people who haven't) | Remove conditional — be direct |
| **Item count mismatch** | "5 items left" in one paragraph, "3" in next | One number throughout |
| **Character age logic** | "Dee childhood Type 2" (Type 2 = adult-onset) | "By the time we were grown" |
| **Inconsistent "queen"** | Mix "queen" and "sis" inconsistently | One term per email |
| **Copy-paste from other email** | Same spreadsheet scene in E1 and E2 | Each email = own imagery |

## 5.5 Proofread Workflow

### Full Audit Process

```
Step 1: READ ALOUD — does it sound like Michelle or a press release?
Step 2: EM DASH COUNT — ≤ 3?
Step 3: LOGIC CHECK — 5 points above
Step 4: AI-ISM SCAN — 8 patterns above
Step 5: QUALITY CHECKLIST — 8 points above
Step 6: ONE FINAL READ — as a COLD READER who's never read an Afroyla email
```

### Audit Output Format

```markdown
## Proofread Audit: [Email Title]

### Summary
- **Em dashes:** [count] found, [count] fixed
- **AI-isms found:** [list]
- **Logic issues:** [list]
- **Grade:** [A+ to C]

### Line-by-Line Fixes
| Line | Issue | Fix |
|------|-------|-----|
| "She paused — and —" | Double em dash | "She paused. And —" |

### Final Assessment
[Clean / Minor fixes / Major rewrite needed]
```

## 5.6 Iteration Patterns (What to Expect)

### Average Iterations by Email Type

| Type | Avg Rounds | Range | Notes |
|------|-----------|-------|-------|
| **Daily email** | 3-5 | 2-7 | First emails take longest |
| **Teaser (individual)** | 3-4 | 2-7 | First in series = hardest |
| **SMS** | 2-3 | 1-6 | SMS voice takes practice |
| **Sale-day email** | 1-2 | 1-3 | Templates speed this up |
| **Post-sale recap** | 1 | 1-2 | Usually near-perfect first draft |

### Common Feedback Patterns

| Feedback | Translation | Action |
|----------|------------|--------|
| "Đọc ko hiểu gì ở đoạn đầu" | Structure problem — flip context first | Move sale info up |
| "Chưa có cao trào" | TELL not SHOW | Add scene with TURN |
| "Hoi giong email trước" | Context block repeated | Rewrite angle/tone |
| "Quá dài, nhiều chi tiết" | Multiple messages crammed | Cut to ONE story |
| "Thiếu tự nhiên" | Voice feels forced | Re-read as friend texting |
| "Ben/Daniel score < 4" | Craft needs work | Apply specific line fixes |
